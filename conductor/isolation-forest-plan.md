# Implementation Plan: Isolation Forest Anomaly Detection

## Objective
Migrate the machine learning anomaly detection system from a Stacked LSTM to an Isolation Forest. This model is better suited for identifying anomalies in the high-entropy, random data generated by the simulator, as it detects statistical outliers based on spatial isolation rather than temporal sequence prediction.

## Key Files & Context
- `ml/train_lstm.py` (To be renamed to `ml/train_anomaly.py`): Current LSTM training script.
- `ml/inference.py`: Current real-time inference consumer.
- `ml/isolation_forest.joblib`: The new saved model file.
- `api/main.py`: Update references if needed.

## Implementation Steps

### 1. Refactor Training Script (`ml/train_anomaly.py`)
- Rename `ml/train_lstm.py` to `ml/train_anomaly.py`.
- Replace the PyTorch Stacked LSTM logic with `sklearn.ensemble.IsolationForest`.
- Features to use: `[valeur, val_mean, val_std]`.
- Save the trained model using `joblib` as `ml/isolation_forest.joblib`.

### 2. Update Inference Script (`ml/inference.py`)
- Remove PyTorch dependencies.
- Load the `ml/isolation_forest.joblib` model.
- Calculate rolling features (`valeur`, `val_mean`, `val_std`).
- Predict the anomaly score (`model.decision_function(X)`).
- Map the threshold logic: The system triggers an alert if the score is below a certain threshold (e.g., `< -0.5`).
- Update the `predictive_alerts` table insertion:
  - `valeur_actuelle` = raw value.
  - `valeur_predite` = anomaly score (instead of predicted value).
  - Update `est_alerte` based on score.
- Output format: `!!! ISOLATION FOREST ALERT [{m_id}] !!! Anomaly detected (Score: {score:.2f})`.

### 3. Verification & Testing
- Run `ml/train_anomaly.py` to ensure the model trains and saves successfully.
- Start the pipeline and verify that `ml/inference.py` correctly consumes Kafka messages and logs alerts.

## Migration & Rollback
- If the Isolation Forest behaves unexpectedly, we can revert `ml/inference.py` and `ml/train_anomaly.py` to the LSTM version.
